> Контекст для передачи проекта другому разработчику.

## Суть задачи
Добавляем **фасетные фильтры** («Подбор») на страницы категорий каталога —
выпадашки характеристик, вытащенных из НАЗВАНИЯ товара регулярками. Раньше
фасеты были только у 5 «жидкостных» категорий, сейчас расширяем на «механику».

## Архитектура (важно: всё data-driven, правится ОДИН файл)
Чтобы добавить категорию — меняется **только** [lib/catalog/attributes.mjs](../lib/catalog/attributes.mjs):
1. `ATTRIBUTE_META[slug]` — список атрибутов: `key`, `label`, `sort`
   (`viscosity` | `numeric` | `list`+`order`).
2. Матчер(ы) — regex по названию + экстрактор в `EXTRACTORS[slug]`.
3. Прогон бэкафилла: `node --env-file=.env.local scripts/backfill-attributes.mjs <slug>`
   — заполняет колонку `products.attributes` (JSONB + GIN-индекс). Идемпотентно,
   печатает % покрытия и топ-значения.

Ни схему БД, ни API, ни UI трогать НЕ нужно — конвейер уже динамический:
`attributes.mjs` → `app/api/catalog/category/[slug]/route.ts` (фасеты+counts через
jsonb `@>`) → `app/catalog/CatalogClient.tsx` (`attr_<key>` в query) →
`components/catalog/CategoryFacets.tsx`. Сайдбар появляется сам при `facets.length > 0`.

## ⚠️ ГЛАВНАЯ ЛОВУШКА — кириллица и `\b`
В JS `\b` (граница слова) работает ТОЛЬКО с ASCII (`\w` = `[A-Za-z0-9_]`).
С кириллицей `\b` **молча не срабатывает**. Уже наступали: `/\bгрм\b/` ловил
37 ремней вместо 3131.
- Для русских слов — подстроки + lookahead: `/задн|зад(?![а-яё])/`, как в `matchVolume`.
- Для латинских кодов (цоколи `H4`/`D2S`, `GAS`/`OIL`, `6PK`) — `\b` корректен.

Переиспользуемый матчер позиции: `matchPosition(name, front, rear)` — формы
передаются («Передний/Задний», «Передние/Задние», «Передняя/Задняя»).

## Что осталось / можно делать дальше
1. Закоммитить `attributes.mjs` (если ещё нет).
2. Задеплоить (авто-деплой по cron на push в `main`) — фасеты на сайте появятся
   только ПОСЛЕ деплоя (в БД данные уже есть).
3. Проверить на живом: `/api/catalog/category/lamps` → в JSON массив `facets`.
4. Ещё лёгкие категории на будущее (reuse `matchPosition`/`matchJointSide`):
   tie-rods, stabilizer-links, ball-joints, silent-blocks (положение/сторона).
   Фильтры воздуха/масла/салона — сложно (размеры в названии нерегулярны).
   `struts` (опоры) в наличии НЕТ — классификатор кладёт их в shock-absorbers/misc.
