## Как компоненты связаны
- **Сайт (VPS) → база VK:** по SSL (`sslmode=require`), порт 5432. Firewall базы пускает **только IP VPS** (`217.114.7.83/32`), остальной интернет отрезан.
- **Сайт → авторизация:** сейчас Supabase (емейлы/пароли). После Фазы 2 — свой GoTrue на VPS, данные в базе VK.
- **Сайт → картинки:** VK Object Storage (S3).
- **Поставщики → почта beget → ночной импорт (cron на VPS) → база VK.**

## Как посмотреть данные в VK (когда понадобится)
В VK **нет веб-редактора таблиц** (в отличие от Supabase Studio). Смотреть строки — через
SQL-клиент на компе: **DBeaver / TablePlus / pgAdmin** (бесплатные), подключение к
`212.233.99.36:5432` (user / пароль из `.env`, `sslmode=require`).
⚠️ Firewall пускает **только VPS**. Чтобы зайти с домашнего компа — два пути:
- **временно** добавить свой IP в группу `SecGroup_brocar-db` (TCP 5432), посмотреть, потом удалить правило;
- **SSH-туннель** через VPS (firewall не трогаем): `ssh -L 5432:212.233.99.36:5432 root@217.114.7.83`, затем клиент подключать к `localhost:5432`.

⚠️ Логины (`auth.users`) **пока на Supabase** — в VK их нет до Фазы 2. В VK сейчас видно: профили, заказы, гараж, каталог.
