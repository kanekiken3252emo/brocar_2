# Уведомление в Роскомнадзор (РКН) об обработке ПДн — BroCar

> ⚠️ Это подготовленный черновик для подачи, **не юридическая консультация**. Перед
> отправкой сверь реквизиты со своими учредительными документами. Содержательные
> разделы заполнены по тому, что реально делает сайт; реквизиты — за тобой.

**Зачем:** с 30.05.2025 уведомление обязательно практически для всех операторов ПДн.
Штраф за неподачу для ИП/юрлиц — **100 000–300 000 ₽** (ч.10 ст.13.11 КоАП).
После переезда БД и авторизации в РФ можно подать **чисто: «всё хранится в РФ,
трансграничной передачи нет»**.

---

## Шаг 0. Проверить — не зарегистрированы ли уже

Реестр операторов: **https://pd.rkn.gov.ru/operators-registry/operators-list/**
Искать по **ИНН** или названию (BroCar / юр.название).

- **Нашёлся** → подаём **КОРРЕКТИРУЮЩЕЕ** (обновление сведений). Главное изменение —
  место хранения теперь РФ, трансграничной передачи нет. (РКН и так просил тех, кто
  подавал до 2023, обновиться по новой форме.)
- **Не нашёлся** → **ПЕРВИЧНОЕ** уведомление.

## Шаг 1. Как подавать

**pd.rkn.gov.ru** → сформировать уведомление → **через Госуслуги** (нужна
подтверждённая учётка **ИП/организации** в ЕСИА) либо подписать **ЭЦП**.
После отправки система выдаст **НОМЕР и КЛЮЧ** — сохрани (понадобятся для правок).

## Раздел «Цели обработки» — ГОТОВО (каждую вносить отдельной целью)

1. **Заключение и исполнение договоров купли-продажи автозапчастей** (оформление и обработка заказов).
2. **Доставка товаров** покупателям.
3. **Регистрация и ведение личного кабинета** пользователя на сайте brocarparts.ru.
4. **Информирование клиентов о статусе заказа** (email-уведомления).
5. **Обеспечение безопасности и предотвращение мошенничества.**

Для каждой цели одинаково:
- **Категории субъектов:** клиенты (покупатели), зарегистрированные пользователи сайта.
  _(+ «работники», если есть наёмные сотрудники — уточнить.)_
- **Категории ПДн:** ФИО; контактные данные (email, телефон); адрес доставки;
  данные учётной записи (email/логин). **Специальных категорий и биометрии — НЕТ.**
- **Правовое основание:** согласие субъекта (п.1 ч.1 ст.6 152-ФЗ) **+** исполнение
  договора (п.5 ч.1 ст.6 152-ФЗ).
- **Перечень действий:** сбор, запись, систематизация, накопление, хранение,
  уточнение (обновление), использование, предоставление, удаление, уничтожение —
  **автоматизированная** обработка.

## Раздел «Трансграничная передача» — ГОТОВО
**НЕ осуществляется.** Все данные хранятся и обрабатываются в РФ (см. ниже).

## Раздел «Меры по обеспечению безопасности» — ГОТОВО

**Организационные:**
- политика в отношении обработки ПДн опубликована на сайте (страница «Политика
  конфиденциальности», /legal/privacy);
- получение согласия субъекта на обработку ПДн при регистрации;
- определение угроз безопасности, внутренний контроль, ограничение доступа к ПДн.

**Технические:**
- передача данных по защищённому каналу **HTTPS/TLS**;
- пароли хранятся в виде **криптографических хешей (bcrypt)**, не в открытом виде;
- **ограничение числа попыток входа** (защита от подбора пароля);
- доступ к базе данных ограничен **межсетевым экраном** (только с сервера приложения);
- **резервное копирование** (managed PostgreSQL, VK Cloud).

_Платёжные данные банковских карт оператор **НЕ хранит** — приём оплаты через
ЮKassa (отдельный оператор/платёжный агрегатор)._

## Раздел «Место нахождения базы данных с ПДн граждан РФ» — ГОТОВО (ключевое для 152-ФЗ)
- База данных размещена на территории **Российской Федерации**: **VK Cloud**
  (managed PostgreSQL), регион **Москва**.
- Сервер приложения (хостинг сайта) — **в РФ** _(уточнить провайдера VPS — впишем)_.
- Трансграничная передача отсутствует.

- **ООО:** должность, ФИО, рабочий телефон + email (будут публичными).

## Что нужно от тебя, чтобы добить
Реквизиты уже заполнены (см. раздел «Сведения об операторе»). Осталось:
1. **Проверка реестра** (Шаг 0): зарегистрирован ИП Бакиров А.О. как оператор — **есть / нет** → первичное или корректирующее.
2. **Подтверждённый Госуслуги ИП** (ЕСИА как предприниматель) — есть? (через него подаётся проще всего).
3. **Хостинг VPS** — провайдер (для строки «сервер приложения в РФ»; БД-то VK Москва, а вот сам сайт где — уточни у того, кто заводил сервер).
4. **Дата начала обработки** — дата регистрации ИП (точную из выписки ЕГРИП).
5. Наёмные сотрудники у ИП есть? (если да — добавим категорию субъектов «работники» + ответственное лицо; если нет — поле ответственного можно не заполнять).
7. **Дата начала обработки** (регистрация бизнеса / запуск сайта).
